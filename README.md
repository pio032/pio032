### ByteBar

An operational management system that coordinates five distinct roles — from table-side ordering to kitchen display to the till — in real time, on any device.

<table>
<tr><th align="left">Feature</th><th align="left">What it does</th></tr>
<tr><td><strong>Role-based auth</strong></td><td>Five session-protected interfaces (Admin, Waiter, Kitchen, Bar, Cashier), each with scoped route permissions</td></tr>
<tr><td><strong>Real-time order flow</strong></td><td>Orders placed at the table auto-refresh on kitchen and bar screens every few seconds without page reload</td></tr>
<tr><td><strong>Availability control</strong></td><td>Products can be toggled out of stock instantly; the change propagates to all active client sessions</td></tr>
<tr><td><strong>Photo upload</strong></td><td>Product images accepted via Multer from both desktop file picker and mobile camera input</td></tr>
<tr><td><strong>Statistics dashboard</strong></td><td>Sales volume, revenue ranking by product, and trend charts rendered client-side with Chart.js</td></tr>
<tr><td><strong>Excel export</strong></td><td>Multi-sheet XLSX reports covering daily sales, product ranking, and full transaction history</td></tr>
</table>

### FlightBook

A flight planning and logging platform that takes you from airport search through route calculation, weather briefing, and AI-assisted analysis — all in a single session.

<table>
<tr><th align="left">Feature</th><th align="left">What it does</th></tr>
<tr><td><strong>Airport search</strong></td><td>Query by ICAO code, IATA code, or partial name; returns full coordinates, country, and municipality</td></tr>
<tr><td><strong>3D route map</strong></td><td>Mapbox GL JS renders departure and arrival on a globe with interactive live-weather markers along the route</td></tr>
<tr><td><strong>Performance calculator</strong></td><td>Distance, estimated flight time, and fuel burn derived from the selected aircraft's stored spec sheet</td></tr>
<tr><td><strong>AI route analysis</strong></td><td>Ollama (gemma3:1b) answers route-specific questions via SSE streaming; every Q&A pair saved per user</td></tr>
<tr><td><strong>Telegram 2FA</strong></td><td>Login triggers a bot message with a one-time code; server verifies it before granting session access</td></tr>
<tr><td><strong>Flight log</strong></td><td>Routes stored with timestamps; monthly and all-time flight hours queryable from the archive</td></tr>
</table>
